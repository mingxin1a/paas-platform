# CRM 细胞 - 部署指南

**版本**：1.2.0

## 1. 部署方式

- **容器**：独立运行在本目录执行 `docker compose up -d`（使用本目录 Dockerfile）；平台联调时由 `deploy/docker-compose.yml` 使用本目录 Dockerfile 构建并接入网关。
- **环境变量**：`PORT`（默认 8001）、与平台约定的 `MONITOR_CENTER_URL`（供 ai_agent 连接监控中心）。

## 2. 网关与路由

- 平台网关将 `/api/v1/crm/*` 转发至 CRM 细胞服务（如 `http://crm-cell:8001/*`）。
- 部署时需在网关或环境中配置 `CELL_CRM_URL` 指向本细胞地址。

## 3. 依赖

- Python 3.10+，Flask。无外部业务库。
- 数据库：当前实现为内存存储；生产可替换为独立 Schema（见 `database_schema.sql`）。

## 4. 验证

- 健康检查：`GET {CELL_URL}/health` 返回 200。
- 全量验收：在项目根执行 `./run.sh verify crm` 或 `./scripts/verify_delivery.sh crm`。
