# 零心智负担与 3-Click 验收清单

**依据**：00 修正案 #12、#13；《01_核心法律》3.1 AI 交互铁律；《接口设计说明书》2.3  
**用途**：对「操作路径深度 ≤3 层」「超过 3 次点击须 AI 一键自动化」建立验收标准并逐项检查。

---

## 一、宪法要求

- **00 #12（自动驾驶原则）**：若操作需要点击超过 3 次，AI 必须提供「一键自动化」方案。
- **00 #13（零心智负担原则）**：任何业务操作路径深度不得超过 3 层；严禁「为点确定需先看 1000 字说明」；AI 须提供意图预判。
- **01 3.1**：管家式 AI、意图预判、零 Prompt 输入。

---

## 二、3-Click 验收标准

| 检查项 | 标准 | 验收方法 |
|--------|------|----------|
| 核心业务操作路径深度 | 从入口到完成主操作（如「创建订单」「入库确认」）≤ 3 次点击 | 人工走查：记录从首页到完成操作的点击次数 |
| 列表→详情→编辑 | 列表 → 详情 → 保存，合计 ≤ 3 层（含列表进入详情算 1 层） | 按细胞逐项走查并记录 |
| 配置/设置入口 | 常用设置（如切换细胞、主题）≤ 2 次点击 | 检查主导航与首屏 |
| 异常/错误恢复 | 错误页提供「返回首页」或「重试」等单一操作，≤ 2 次点击回到正常路径 | 模拟错误场景走查 |

---

## 三、按模块验收表（示例）

| 模块 | 操作 | 当前点击深度 | 是否 ≤3 | 备注 |
|------|------|--------------|--------|------|
| 客户端-概览 | 进入控制台 | 1（登录后即概览） | 是 | — |
| 客户端-细胞列表 | 进入某细胞列表 | 2（概览→细胞） | 是 | — |
| 客户端-细胞详情 | 列表→详情 | 3 | 是 | — |
| 管理端-细胞管理 | 启用/停用细胞 | 2～3 | 是 | 细胞列表→操作 |
| 管理端-用户权限 | 修改用户 allowedCells | 3 | 是 | — |
| 管理端-审计日志 | 查看审计 | 2 | 是 | — |

---

## 四、AI 一键自动化与意图预判（规划）

| 能力 | 要求 | 当前状态 | 验收标准 |
|------|------|----------|----------|
| 一键自动化 | 超过 3 次点击的操作提供一键入口 | 规划中 | 产品定义高频多步操作并提供一键入口 |
| 意图预判 | 用户输入时系统已准备下一步 | 规划中 | AI 或前端根据上下文预加载/预填 |
| 零 Prompt | 禁止用户输入复杂 Prompt | 规划中 | AI 交互采用选择/推荐而非长文本输入 |

---

## 五、使用说明

- 每次发布前或合规自检时，按本清单对新增/变更业务路径做 3-Click 走查。
- 若某操作超过 3 次点击，须在需求/设计中补充「一键自动化」或缩短路径的方案。
- 与《宪法符合性检查与不符合项清单》不符合项 #3 对应。
