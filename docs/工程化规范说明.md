# 全平台工程化规范说明

## 1. 代码风格统一

- **Python**：缩进 4 空格，UTF-8，LF。见 **.editorconfig**。
- **Vue/TS/JS**：缩进 2 空格。见 **.editorconfig**。
- 各仓库根目录可放置 `.editorconfig` 与 Lint 配置（如 ESLint、Ruff/Black），与本文档保持一致。

## 2. 核心代码注释

- 目标：核心代码注释覆盖率 ≥80%（模块/类/关键函数说明）。
- 已做：网关 `platform_core.core.gateway.app` 中 `create_app`、`proxy`、错误响应等已补充注释；限流、审计、敏感字段模块带模块与函数注释。
- 建议：各细胞按优先级对 `app.py`、`store.py` 及对外接口逐文件补全 docstring，再运行注释覆盖率工具校验。

## 3. 冗余代码与测试文件清理

- 建议按细胞分批：删除未使用函数、废弃路由、重复工具函数；合并或移除重复/过时测试用例，保留核心流程与跨细胞协作测试。
- 不修改核心业务逻辑；仅做删除与合并，避免破坏细胞化架构。

## 4. 文档与 API

- 统一交付包结构见 **《商用交付包目录结构》**。
- 细胞独立交付文档见 **《细胞独立交付文档索引》**。
- API 与错误码见 **《错误码规范》**、**docs/api/openapi.yaml**、**docs/api/error_codes.md**；OpenAPI 可导入 Postman 使用。
