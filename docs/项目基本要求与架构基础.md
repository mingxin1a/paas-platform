# 项目基本要求与架构基础

本文档为本仓库**统一入口**：明确什么是不可违背的基本要求，以及架构与设计以何为基础、如何扩展。

---

## 一、基本要求：不允许违背

以下两份文档为本项目**基本要求**，**一切设计与实现均不允许违背**：

| 文档 | 说明 |
|------|------|
| **00_【最高宪法】SuperPaaS-God** | 五重审判、人性审判（≤3 次点击、零心智负担）、禁止 Lombok/反射/动态代理、金融用 Long/BigDecimal、JSON 日志与 trace_id（CT 扫描）、集装箱/玻璃房等修正案。 |
| **01_【核心法律】基础与AI安全宪法** | 技术栈与分层（合规为 Java 17+、Spring Boot 3+、DDD）；细胞自治、跨细胞仅事件/接口且必须异步、平台不持有业务状态；多租户、敏感字段加密、全链路 trace_id、数字签名与验签等。 |

- **裁决原则**：与需求、工期或其它文档冲突时，**以 00/01 为准**。
- **落地细则**：技术选型与构建条款见 `docs/00-01_构建合规说明.md`。

---

## 二、架构与设计：以文档为基础，可扩展

**架构及相关设计**以本仓库内**设计文档**为基础，可在此基础上**扩展**。

### 2.1 作为基础的文档（示例）

- 《**超级PaaS平台全量化系统架构设计说明书**》—— 五层架构、感知交互层、细胞化、安全与运维等。
- 《02_概要设计说明书》《接口设计说明书》《数据库设计说明书》《详细设计说明书》《超级PaaS平台部署方案》等平台层文档。

### 2.2 扩展方式

- 一切以**文档为最高要求**，**只允许在文档基础上扩展**。
- 可在文档基础上**新增子模块、细化某层实现、补充接口与数据规范**等。
- **扩展内容不得与 00/01 相悖**；若有冲突，以 00/01 为准。
- 不得引入与文档或 00/01 相悖的**重复、无用或冗余**实现；存在多套实现时须在文档中明确以何者为合规基准。

### 2.3 文档与层次关系

- **战略层**（00/01）：基本要求，不可违背。
- **平台层**（全量化、概要/接口/数据库/详细设计、部署等）：架构与设计的基础，可扩展。
- **应用层**（各业务细胞）：在平台规范与 00/01 约束下实现与扩展。

---

## 三、小结

| 维度 | 约定 |
|------|------|
| **基本要求** | 00 最高宪法、01 核心法律 —— **不允许违背** |
| **架构与设计** | **以文档为基础**（全量化等），**可在此基础上扩展** |
| **冲突时** | 以 00/01 为裁决依据 |
