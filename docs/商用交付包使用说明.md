# 商用交付包使用说明

本文档说明全项目标准化商用交付包的内容、生成方式、部署步骤、验收标准与售后支持。

---

## 一、交付包生成

执行一键构建脚本后，将在 `dist/` 下生成标准化交付目录及压缩包：

```bash
python scripts/build_full_delivery.py
```

生成物：

- **目录**：`dist/SuperPaaS_Delivery_YYYYMMDD/`
- **压缩包**：`dist/SuperPaaS_Delivery_YYYYMMDD.zip`

---

## 二、交付包内容

| 目录 | 说明 |
|------|------|
| **01_平台核心包** | PaaS 核心层代码（platform_core）、部署脚本（run_gateway 等）、部署配置、运维工具（scripts 下自检/校验等） |
| **02_分模块交付包** | 各业务 Cell 独立包（cell_crm、cell_erp、cell_oa 等），每包含：代码、Dockerfile/部署配置、接口文档、用户手册 |
| **03_文档包** | 总交付手册、架构文档、部署运维手册、用户操作手册、管理员手册、合规文档、验收清单 |
| **04_测试包** | 全量测试用例、测试脚本、测试报告、验收测试工具（self_check、verify_all_cells 等） |
| **05_部署工具包** | 一键部署脚本、docker-compose、k8s 配置、环境配置示例（.env.example） |
| **商用交付包使用说明.md** | 本说明（部署步骤、验收标准、售后支持） |

---

## 三、部署步骤

1. **环境准备**：Python 3.8+、Node 18+（前端）、Docker（可选）。
2. **平台核心**：从 `01_平台核心包/deploy` 执行 `python run_gateway.py`，网关默认端口 8000；需配置细胞路由（环境变量 `CELL_*_URL` 或 `GATEWAY_ROUTES_PATH`）。
3. **细胞部署**：从 `02_分模块交付包/cell_<id>` 按各细胞 README 或 Dockerfile 部署，并确保网关可访问细胞地址。
4. **前端**：从项目根或交付包外获取 frontend、frontend-admin，执行 `npm install && npm run build`，可通过网关静态目录托管或独立部署。
5. **验收**：使用 `04_测试包/验收工具` 及 `03_文档包/验收清单` 执行验收。

详细步骤见 `03_文档包/部署运维手册` 及 `03_文档包/总交付手册`。

---

## 四、验收标准

- **可用性**：网关 `/health` 返回 200；各细胞经网关 `/api/v1/<cell>/health` 可达。
- **业务流程**：登录、列表、创建等核心流程按《商用验收清单》及《批次1_商用验收测试用例》通过。
- **安全与合规**：按《安全合规说明》《安全渗透测试报告》及验收清单执行。
- **性能**：按《全平台性能测试报告》及性能测试计划执行（可选）。

---

## 五、售后支持说明

- **文档与问题清单**：见交付包内 `03_文档包` 及项目 `docs/`。
- **缺陷与需求反馈**：通过项目 Issue 或商务约定渠道反馈。
- **版本与补丁**：以交付包版本号（SuperPaaS_Delivery_YYYYMMDD）及发布说明为准。
