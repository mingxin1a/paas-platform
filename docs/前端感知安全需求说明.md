# 前端感知安全需求说明（00 修正案 #14 / 01 4.1）

**依据**：《00_最高宪法》修正案 #14 玻璃房原则、《01_核心法律》4.1 感知安全。  
**适用范围**：所有面向用户的前端（Web/移动端）在实现时须满足以下要求。

---

## 1. 安全态势可视化

- 在**登录、支付、敏感操作**时，须提供「安全态势可视化」体验，让用户感知到安全，例如：
  - 登录时：显示登录地点地图、设备信息、本次登录时间。
  - 支付/敏感操作时：显示数据正在「国密盾」或加密通道中传输的动画或提示。
- 安全不能牺牲体验，但要让用户「感觉」到安全。

## 2. 动态水印

- 所有界面须**实时生成包含用户名、IP、时间的水印层**（01 4.1 动态水印），防止截屏泄密与责任追溯。

## 3. 透明化审计

- 用户须能**随时查看「谁在什么时候访问了我的数据」**（00 #14、01 4.1 透明化审计）。
- 前端需提供入口（如「我的数据访问记录」）展示与 trace_id 关联的访问日志。

---

## 4. 当前状态

- 已提供**统一前端控制台**（`frontend/`）：React + Vite + TypeScript，对接网关，含概览、细胞列表/详情、明暗主题、动态水印与审计入口占位。
- **前端实现时**须将上述三项（安全态势可视化、动态水印、透明化审计）纳入需求与验收标准，不得遗漏；当前水印与审计为占位，生产需对接真实数据。
