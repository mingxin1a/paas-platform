# 超级PaaS平台商用验收清单

**版本**：1.0  
**用途**：交付前逐项核对，满足商用交付标准；验收人可勾选确认。  
**使用**：在「状态」列标注 通过 / 未通过 / 不适用。

---

## 一、功能商用化

| 序号 | 检查项 | 标准 | 状态 |
|------|--------|------|------|
| F1 | 各 Cell 核心业务流程闭环 | 采购/销售/库存/财务等流程端到端可跑通 | |
| F2 | 最小商用集覆盖 | ERP/CRM/MES 等覆盖领域最小商用功能集 | |
| F3 | 业务配置能力 | 审批流程、字段自定义、报表配置至少一种 | |

---

## 二、数据安全商用化

| 序号 | 检查项 | 标准 | 状态 |
|------|--------|------|------|
| S1 | 多租户数据隔离 | PaaS + Cell 内均按租户隔离 | |
| S2 | 传输加密 | 生产环境 HTTPS | |
| S3 | 存储加密与脱敏 | 敏感字段加密或脱敏展示 | |
| S4 | RBAC 权限 | 菜单/按钮/数据级权限，可配置角色 | |
| S5 | 操作日志与审计 | 谁/何时/做了什么，可查且不可篡改 | |

---

## 三、性能商用化

| 序号 | 检查项 | 标准 | 状态 |
|------|--------|------|------|
| P1 | 核心接口响应时间 | 单机 P95 小于等于 500ms | |
| P2 | 并发能力 | 支持 100+ 并发用户 | |
| P3 | 数据库索引 | 无全表扫描，热点有索引 | |
| P4 | 批量操作 | 支持批量导入/导出或批量审核 | |

---

## 四、可靠性商用化

| 序号 | 检查项 | 标准 | 状态 |
|------|--------|------|------|
| R1 | 异常处理体系 | 参数/业务/系统分层，错误信息商用级 | |
| R2 | 幂等性 | 核心写接口支持 X-Request-ID 幂等 | |
| R3 | 健康检查 | 所有服务提供 /health，可被探测 | |

---

## 五、可运维商用化

| 序号 | 检查项 | 标准 | 状态 |
|------|--------|------|------|
| O1 | Docker/K8s 部署 | 提供完整部署/升级/回滚方式 | |
| O2 | 监控 | Prometheus/Grafana 或等价监控 | |
| O3 | 备份与恢复 | 数据备份/恢复脚本（全量+增量） | |
| O4 | 日志标准化 | 分级、可检索、ELK 兼容 | |

---

## 六、合规商用化

| 序号 | 检查项 | 标准 | 状态 |
|------|--------|------|------|
| C1 | 接口规范 | 遵循《接口设计说明书》，OpenAPI 文档完整 | |
| C2 | Cell 交付手册 | 每 Cell 具备部署/运维/使用/故障处理文档 | |
| C3 | 代码规范 | PEP8/ESLint，注释覆盖率不小于 80% | |

---

## 七、解耦架构（红线）

| 序号 | 检查项 | 标准 | 状态 |
|------|--------|------|------|
| A1 | PaaS 无业务逻辑 | 平台核心仅通用能力 | |
| A2 | Cell 独立部署 | 独立库/代码/部署，仅经网关交互 | |
| A3 | 变更隔离 | 新增/改 Cell 不影响 PaaS 及其他 Cell | |
