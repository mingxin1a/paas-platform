# WCAG 2.1 无障碍验收清单

**依据**：00 修正案 #16、01 要求；《详细设计说明书》2.无障碍功能测试  
**用途**：支持读屏软件与键盘操作，满足视障与行动障碍用户使用；按 WCAG 2.1 AA 做自检与文档化。

---

## 一、宪法要求

- **00 #16**：系统必须支持无障碍访问（视障人士也能通过读屏软件使用 ERP）。
- **01**：体验与安全专版中感知交互层包含无障碍支持。

---

## 二、WCAG 2.1 AA 自检要点

| 原则 | 检查项 | 验收方法 | 当前状态 |
|------|--------|----------|----------|
| 可感知 | 图片/图标有替代文本（alt / aria-label） | 检查所有 img 与图标按钮 | 部分：Layout 主题/导航有 aria-label |
| 可感知 | 表单控件有可见或关联的标签 | 检查 input 与 label / aria-labelledby | 按页检查 |
| 可操作 | 键盘可访问：Tab 顺序合理、焦点可见 | 仅用键盘完成主要流程 | 待全量测试 |
| 可操作 | 无键盘陷阱；可跳过重复导航 | 焦点不困在模态/侧栏 | 待测试 |
| 可理解 | 错误提示与说明清晰 | 错误信息与 label 易读 | 按页检查 |
| 健壮性 | 语义化 HTML 与 ARIA 正确 | 按钮/链接/区域 role 正确 | 部分：SecurityStatus 使用 role=region |

## 三、前端实施要点

---

## 四、读屏与键盘测试

- **读屏**：使用 NVDA / VoiceOver 等，遍历主要流程（登录、概览、细胞列表、详情），确认关键操作可被朗读且无冗余。
- **键盘**：仅用 Tab / Enter / Space / Esc 完成登录、导航、退出；确认焦点可见（:focus-visible）且无陷阱。
