# 通用制造业解决方案 · 方案架构

**解决方案**：通用制造业  
**模块组合**：ERP + MES + WMS + SRM + TMS  

---

## 一、架构原则

- **细胞化架构**：各业务模块为独立 Cell，仅经 PaaS 网关标准化接口交互，无直接依赖。
- **按需选配**：制造方案默认包含 ERP、MES、WMS、SRM、TMS；客户可减少或增加模块（如仅 ERP+WMS），不修改平台核心。
- **统一入口**：所有请求经网关认证、路由、审计后转发至对应 Cell，租户与权限由平台统一管理。

---

## 二、逻辑架构图（文字描述）

```
                    [ 前端 / 移动端 ]
                            |
                    [ PaaS 网关 · 认证/路由/审计 ]
                            |
        +--------+--------+--------+--------+--------+
        |        |        |        |        |        |
      [ ERP ] [ MES ] [ WMS ] [ SRM ] [ TMS ]
        |        |        |        |        |
        +--------+--------+--------+--------+
                    |
            [ 统一租户与权限 ]
```

- **ERP**：计划、采购、销售、库存账、成本、财务主线。
- **MES**：工单执行、报工、BOM/工艺、生产实绩。
- **WMS**：实物入库、上架、拣配、出库、盘点、批次。
- **SRM**：供应商、询报价、订单协同、对账。
- **TMS**：运单、承运商、在途、签收。

---

## 三、数据流与集成点

| 场景           | 主流程                         | 涉及 Cell    |
|----------------|--------------------------------|--------------|
| 计划到生产     | ERP 工单 → MES 接收执行 → 报工回写 | ERP、MES     |
| 采购到入库     | SRM/ERP 采购订单 → 到货 → WMS 入库 | ERP、SRM、WMS |
| 生产领料       | MES 领料需求 → WMS 拣配出库       | MES、WMS     |
| 销售出库与发运 | ERP 销售出库 → WMS 拣配出库 → TMS 运单 | ERP、WMS、TMS |

所有跨 Cell 数据通过 PaaS 网关标准 API 调用，无 Cell 间直连。

---

## 四、部署架构建议

- **最小部署**：1 台网关 + 5 个 Cell 进程（或容器），可同机或分机。
- **生产建议**：网关独立、Cell 按负载拆分；数据库可按 Cell 独立实例或 Schema 隔离。
- **扩展**：新增 PLM、EMS 等时，仅新增 Cell 与路由配置，不改已有 Cell 与网关核心。
