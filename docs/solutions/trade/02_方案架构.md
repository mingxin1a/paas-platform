# 通用贸易企业解决方案 · 方案架构

**模块组合**：CRM + ERP + WMS + SRM + OA  

---

## 一、架构原则

- **细胞化架构**：各业务模块为独立 Cell，仅经 PaaS 网关交互，无直接依赖。
- **按需选配**：贸易方案默认 CRM、ERP、WMS、SRM、OA；可减少或增加模块（如加 TMS），不修改平台核心。
- **统一入口**：所有请求经网关认证、路由、审计后转发至对应 Cell。

---

## 二、逻辑架构

```
                [ 前端 / 移动端 ]
                        |
                [ PaaS 网关 ]
                        |
    +--------+--------+--------+--------+--------+
    |        |        |        |        |        |
  [ CRM ] [ ERP ] [ WMS ] [ SRM ] [ OA ]
```

- **CRM**：客户、商机、合同、回款、报表。
- **ERP**：采购、销售、库存、成本、应收应付。
- **WMS**：入库、出库、库存、库位、批次。
- **SRM**：供应商、询报价、订单协同、对账。
- **OA**：审批、公告、任务、流程。

---

## 三、数据流与集成点

| 场景 | 主流程 | 涉及 Cell |
|------|--------|-----------|
| 销售到出库 | CRM 合同/订单 → ERP 销售订单 → WMS 出库 | CRM、ERP、WMS |
| 采购到入库 | SRM/ERP 采购订单 → 到货 → WMS 入库 | ERP、SRM、WMS |
| 审批与业务 | OA 审批单与 ERP/CRM 业务单据关联（可扩展） | OA、ERP、CRM |

---

## 四、部署架构建议

- **最小部署**：1 台网关 + 5 个 Cell 进程（或容器）。
- **扩展**：新增 TMS、PLM 等时，仅新增 Cell 与路由配置。
