# 医疗行业解决方案 · 方案架构

**模块组合**：HIS + LIS + OA + ERP  

---

## 一、架构原则

- **细胞化架构**：各业务模块为独立 Cell，仅经 PaaS 网关交互，无直接依赖。
- **按需选配**：医疗方案默认 HIS、LIS、OA、ERP；可减少或增加模块（如仅 HIS+LIS），不修改平台核心。
- **统一入口**：所有请求经网关认证、路由、审计后转发至对应 Cell；医疗数据合规由各 Cell 与平台审计共同保障。

---

## 二、逻辑架构

```
                [ 前端 / 移动端 ]
                        |
                [ PaaS 网关 · 认证/路由/审计 ]
                        |
    +--------+--------+--------+--------+
    |        |        |        |        |
  [ HIS ] [ LIS ] [ OA ] [ ERP ]
```

- **HIS**：患者、挂号、就诊、处方、收费、住院、病历、审计。
- **LIS**：检验申请、样本、结果、报告、审核、发布、审计。
- **OA**：审批、公告、任务、流程。
- **ERP**：采购、库存、成本、应收应付（药品/耗材等）。

---

## 三、数据流与集成点

| 场景 | 主流程 | 涉及 Cell |
|------|--------|-----------|
| 就诊与检验 | HIS 就诊/处方 → LIS 检验申请 → 样本→结果→报告 → HIS 结果查阅 | HIS、LIS |
| 物资与领用 | ERP 采购/库存 → HIS 或 LIS 领用申请（可经 OA 审批） | ERP、HIS、LIS、OA |
| 审批与业务 | OA 审批与 HIS/ERP 业务单据关联（可扩展） | OA、HIS、ERP |

---

## 四、部署架构建议

- **最小部署**：1 台网关 + 4 个 Cell 进程（或容器）。
- **合规**：医疗数据存储与传输需满足等保及行业要求；审计日志不可篡改、可追溯。详见《医疗行业合规适配说明》。
